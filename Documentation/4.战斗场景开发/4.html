    🟡自制攻击效果¶

    也许你已经对很常见的木棍、真刀等效果产生了审美疲劳。没关系， SoulEngine 支持自定义攻击样式，所有的攻击效果，都由你自己亲手制作。

    攻击文件的位置¶

    那么在先前我们已经提到过一个战斗可以自定义攻击样式，而它们的位置都位于 Scripts/Libraries/Battle/Patterns/ 文件夹中。

    里面拥有三个文件，一个是 doubleatk ，是先前用于构建双重攻击的文件；第二个是 stick ，是经典的木棍单刀攻击；最后一个是 template ，主要用于复制修改的底层模板。

    攻击的运作顺序¶

    接下来我们详解一下攻击这个动作底层的逻辑顺序，以便于更好的理解底层是怎么工作的。

    首先，玩家进入 ATTACKING 状态，在这个状态，会触发攻击库的第一层函数 Create() 。在这里，你需要初始化战斗框里面的UI，比如如果是原版那么创建一个校准器并且创建一个校准条；如果是模拟 Undertale Yellow 这款游戏 Clover 的攻击方式，那么需要先创建左轮手枪的六发弹夹截面。

    其次，在创建之后，所有逻辑均由 Update 函数掌管，包括攻击条什么时候动、怎么动、什么时候停、怎么计算伤害，或者玩家可以连按几次确认来确认攻击位置、平地锅等多校准条武器，内部怎么控制的完全由你自行编写 Update 的逻辑。

    最后，计算总伤害并交由 finalHurt ，让 finalHurt 进行后续一系列的处理操作。

    下面是全过程的整理：

    STATE 攻击动画 行为描述
    刚进入ATTACKING 调用Create函数 所有战斗UI或条件判定进行初始化
    ATTACKING 持续调用Update函数 自定义战斗样式持续更新
    ~ATTACKING（无明确定义，但一般是ATTACKING） BattleInit持续等待_end变量结束攻击，计算伤害并反馈 战斗系统底层会等待_end被设置为true，推进游戏进程
    ~（任何状态均可，除ATTACKING） 调用Reset函数，重置所有攻击参数和变量，为下一次攻击做准备 攻击效果回收，变量初始化，回收所有动画

    所以综上来看，我们只需要修改这三部分内容即可制作自定义攻击动画和效果。

    制作攻击动画¶

    看了上面的内容，也许你对逻辑有一个大概的认识了，接下来让我们着手写一个自定义攻击动画。

    底层变量¶

    这些变量均不可以被省略，必须进行设置，因为更底层的系统需要它们有一个确切的值。

    • enemy：每一个战斗动画都会需要这个enemy进行获取敌人信息，包括你怎么知道敌人位置的，都在这个变量里面。
    • _end：如果设置为了 true ，那么底层战斗系统会立刻更新弹出字体的逻辑，请只在确保攻击真的结束之后设置为 true 。例如如果制作单刀效果（如木棍、真刀等），则在按下确认键定住校准条之后设置为 true 。
    • finalHurt：在 _end 之前设置 finalHurt 。如果不想给敌人造成伤害请填写负数来制作出 MISS 的效果。

    其余所有变量均为自己创建（参考 template 文件中的内容）。

    解读默认攻击模板¶

    也许你想知道默认攻击模板做了什么然后拿来复制使用，但是又不知道这里面写了什么？

    没关系，让我们一条一条拆解。

    首先，去掉Create前面的内容，只看Create： 

    function template.Create()
    template._has_sliced = false
    template.finalHurt = 0
    template.Elements = {}

    -- target（示例）
    local target = sprites.CreateSprite("UI/Battle Screen/spr_target_0.png", 12)
    target.y = 320
    target:MoveTo(320, target.y)
    table.insert(template.Elements, target)

    -- moving bar（示例）
    local bar = sprites.CreateSprite("UI/Battle Screen/Player Attack/spr_targetchoice_0.png", 16)
    bar.y = 320
    bar:SetAnimation({
        "UI/Battle Screen/Player Attack/spr_targetchoice_1.png",
        "UI/Battle Screen/Player Attack/spr_targetchoice_0.png"
    }, 5)
    -- 随机左右出发，保持与原逻辑兼容
    local pos = math.random(1, 2)
    if pos == 1 then
        bar.x = 320 + 280
        bar.velocity.x = -6
        bar.newPosvar = 2
    else
        bar.x = 320 - 280
        bar.velocity.x = 6
        bar.newPosvar = 1
    end
    table.insert(template.Elements, bar)
end
    解释：

    它会瞬间创建一个target，也就是我们的校准面板，同时创建一个bar，这是我们的校准条。

    在校准条创建的时候，有一个 pos 变量随机选择这个校准条的位置，如果是1那么它就从右边过来；反之左边。

    这两个图片都会被添加到 template.Elements 表中，便于 Update 函数的后续控制。

    下面我们来看Update： 

    function template.Update(dt)
    -- 如果没有 enemy（极端情况），直接返回
    if not template.enemy then return end

    -- 如果玩家按下 confirm（这里只用示例的 keyboard.GetState，和 Battle 一致）
    if keyboard.GetState("confirm") == 1 and not template._has_sliced then
        template._has_sliced = true
        template._end = true
        print("ATTACKED BY LIBRARY")

        -- 播声音并生成 slice 特效，放在 enemy 的当前位置
        audio.PlaySound("snd_slice.wav")
        local slice = sprites.CreateSprite("UI/Battle Screen/Player Attack/spr_slice_o_0.png", 18)
        slice:SetAnimation({
            "UI/Battle Screen/Player Attack/spr_slice_o_1.png",
            "UI/Battle Screen/Player Attack/spr_slice_o_2.png",
            "UI/Battle Screen/Player Attack/spr_slice_o_3.png",
            "UI/Battle Screen/Player Attack/spr_slice_o_4.png",
            "UI/Battle Screen/Player Attack/spr_slice_o_5.png"
        }, 10, "oneshot-empty")

        -- 使用实时 enemy 位置
        local e = template.enemy
        if e and e.position then
            slice:MoveTo(e.position.x, e.position.y)
        else
            slice:MoveTo(320, 320) -- 回退到中心，避免崩溃
        end
        table.insert(template.Elements, slice)

        -- 计算伤害（示例：基于 bar 与 target 的距离）
        local damage = template.GetMaxDamage() or 0
        local tar = template.Elements[1]
        local bar = template.Elements[2]
        bar.velocity.x = 0  -- 停止移动
        if tar and bar and bar.x and tar.x then
            local absLength = math.abs(bar.x - tar.x)
            if absLength > 5 then
                damage = math.ceil(damage * 0.9 * (1 - absLength / 280))
            end
        end

        -- 将最终伤害写回 pattern，Battle 将在结算时使用它
        template.finalHurt = damage
    end

    -- 检测 bar 越界，判定为不攻击，直接输出 0 伤害
    local bar = template.Elements[2]
    if bar and (bar.x < 40 or bar.x > 600) and not template._has_sliced then
        template._has_sliced = true
        template._end = true
        template.finalHurt = -999
        bar.velocity.x = 0
    end
end
    解释：

    如果玩家按下了攻击，并且是第一次按下（由_has_sliced 变量控制），那么创建一个砍击的效果。并且告诉战斗系统，_end变量已经设置为true了，玩家无法进行任何操作，可以进行结算，并进入下一个回合。

    利用enemy准确获取敌人位置，将攻击特效放到敌人身上。

    然后停住bar，计算bar位置的伤害，最后将伤害交给 finalHurt 变量，来推动战斗系统底层进行工作。

    随后处理玩家一直不攻击的回收系统，在合适的时间点，直接设置 _end 为 true 即可。
