回合

为了方便使用者不需要自己再从头写一些很常用的库，这里列出了本模板所带的所有攻击库及其使用方法以及案例。
方便使用者使用或根据现有库进行新的攻击库的制作。

战斗中的每一个回合的说明都在此处

骨头库(Bones.lua)

全部主函数

  • Bones:New2D(whose, length)
    • whose - 谁的骨头（目前仅Sans和Papyrus两种可用）。
    • length - 初始创建的长度，若不确定长度则可以写0，后续可以进行进一步的更改。
  • Bones:New3D(points)
    • points - 用于确定该立体图形的所有节点的位置，后续可以进行节点连接。
示例： 
                
-- 创建一个长度为10的Sans样式的骨头。
local bone = Bones:New2D("Sans", 10)

-- 创建一个骨立方框架（暂时没有进行连接）。
-- 该代码划定的骨立方区域为以(320, 240, 100)为中心点的立方体，棱长为100.
local bone3d = Bones:New3D({
    {220, 140, 50}, {420, 140, 50}, {220, 340, 50}, {420, 340, 50},
    {220, 140, 150}, {420, 140, 150}, {220, 340, 150}, {420, 340, 150}
})
2D的骨头和3D的骨头各自有不同的方法：
  • 2D骨头
    • bone:SetStencils(masks) - 设置骨头的遮罩，详见贴图文档，方法一致。
    • bone:Length(length) - 设置骨头的长度（一般情况下为了方便也可以直接使用bone.length = x进行设置）。
    • bone:Destroy() - 骨头的自我清除。
    • bone:SetColor(color) - 设置骨头的颜色（color为一个表）。
    • bone:SetMode(mode) - 设置骨头的HurtMode（详见贴图文档，HurtMode的值一致）。
    • bone:ToUp(arena) - 设置骨头贴住框的上边（若变长度则需要重新使用）。
    • bone:ToDown(arena) - 设置骨头贴住框的下边（若变长度则需要重新使用）。
    • bone:ToLeft(arena) - 设置骨头贴住框的左边（若变长度则需要重新使用）。
    • bone:ToRight(arena) - 设置骨头贴住框的右边（若变长度则需要重新使用）。
  • 3D骨头
    • bone3d:Link(whose, point_1, point_2) - 连接3D骨头的两个点，生成一个新的2D骨头对象。
    • bone3d:Rotate(axis, angle) - 围绕指定轴（"x"、"y"、"z"）旋转骨头结构。
    • bone3d:MoveToRelative(x, y) - 设置骨头的相对移动位置。
    • bone3d:MoveTo(x, y) - 移动骨头到指定的绝对坐标。
    • bone3d:SetStencils(masks) - 设置骨头及其所有子骨头的遮罩。
    • bone3d:Destroy() - 销毁骨头对象及其所有子骨头。
示例： 
                
-- 创建一个2D骨头（Sans，长度为80）
local bone2d = Bones:New2D("Sans", 80)

-- 设置遮罩
bone2d:SetStencils({"mask1.png"})

-- 设置长度
bone2d:Length(120)

-- 设置颜色
bone2d:SetColor({1, 0.8, 0.8, 1})

-- 设置HurtMode
bone2d:SetMode("Blue")

-- 贴住框的上边
bone2d:ToUp(arena)

-- 需注意，若想要在改变长度的同时贴住框，则需要先变长度再贴框，顺序不可颠倒.
bone2d.logic = function(self)
    self.length = self.length + 1
    self:ToDown(arena)
end

-- 错误示范：
bone2d.logic = function(self)
    self:ToDown(arena)
    self.length = self.length + 1
end



-- 创建一个3D骨头（立方体8点）
local bone3d = Bones:New3D({
    {220, 140, 50}, {420, 140, 50}, {220, 340, 50}, {420, 340, 50},
    {220, 140, 150}, {420, 140, 150}, {220, 340, 150}, {420, 340, 150}
})
-- 连接1, 2两个点生成2D骨头
bone3d:Link("Papyrus", 1, 2)
bone3d:Rotate("y", math.rad(45))
bone3d:MoveTo(320, 240)
bone3d:SetStencils({mask})
bone3d:Destroy()

GB库(GasterBlaster.lua)

全部主函数

  • GB:New(begin, final, angles, waittime, firingtime, sprite_prefix, sounds_prefix)
    • begin - GB的起始位置
    • final - GB的终点位置
    • angles - GB的起始角度和最终角度
    • waittime - 距离GB从被创建到开火的等待时间（建议大于等于40）
    • firingtime - GB的开火时长
    • sprite_prefix - GB的贴图文件夹路径
    • sounds_prefix - GB的音效文件夹路径
                
-- 创建一个GB，起始位置为(0, -100)，最终目标位置为(320, 240)
-- 起始角度为180，目标角度为0
-- 从创建开始等待40帧后开火60帧

local gb = GB:New(
    {0, -100}, {320, 240}, 
    {180, 0}, 

    40, 60
)
由于一个GB并不是在回合文件内初始化的，所以在回合结束后该GB若被创建出来仍会按照原先逻辑执行。

战斗框库(Arenas.lua)

全部主函数

  • arenas.Init() - 游戏战斗场景的主战斗框的初始化。
  • arenas.NewArena(x, y, w, h, r, shape, mode)
    • x - 战斗框的x
    • y - 战斗框的y
    • w - 战斗框的初始宽度（w >= 16）
    • h - 战斗框的初始高度（h >= 16）
    • r - 战斗框的初始角度
    • shape - 战斗框的图形（目前只有"rectangle"可以使用）
    • mode - 战斗框的模式（"Plus", "Minus"）
                
local arena1 = arenas.NewArena(320, 320, 130, 130, 0, "rectangle")

-- 回合结束前使用
arenas.clear()

回合结束

使用EndWave函数来直接跳出一个回合，若没有EndWave那么该回合将无限执行下去。
