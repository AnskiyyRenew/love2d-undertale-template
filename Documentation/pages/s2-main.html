游戏的主体

对于LOVE2D而言，整场游戏只有一个lua环境，所以这里，就是关于这一个大环境的介绍。
文件位置：main.lua

变量的传递

在main.lua中，所有未加local字眼的变量均为全局变量，在其他任何地方（除了conf.lua）都能被使用、检测。 

                
global = require("Scripts.Libraries.GlobalVariables")
maths = require("Scripts.Libraries.Utils.Mathematics")
collisions = require("Scripts.Libraries.Collisions")
keyboard = require("Scripts.Libraries.Keyboard")
masks = require("Scripts.Libraries.MaskManager")
audio = require("Scripts.Libraries.AudioManager")
layers = require("Scripts.Libraries.Utils.Layers")
sprites = require("Scripts.Libraries.SpritesManager")
typers = require("Scripts.Libraries.TyperManager")
scenes = require("Scripts.Libraries.SceneManager")
tween = require("Scripts.Libraries.Tween")
windows = require("Scripts.Libraries.Utils.Windows")

-- Canvas setup
CANVAS = love.graphics.newCanvas(
    love.graphics.getWidth(),
    love.graphics.getHeight(),
    nil,
    {
        format = "stencil",
        readable = true
    }
)
CANVAS:setFilter("nearest", "nearest")
如上，所有的变量均为全局变量，你可以在任何场景，任何回合代码中使用他们。

独立场景的管理

下面是关于游戏初始化布置场景的地方。 

                
-- Main Love2D callbacks
function love.load()
    -- System setup
    -- 此处是用于控制台能够正常打出中文字符的
    if (not _RELEASED) then
        if (love.system.getOS() == "Windows") then
            local handle = io.popen("chcp 65001", "r")
            handle:close()
        end
    end

    -- Initialization
    -- 这里是背景的初始化，以及游戏时间的开端记录
    startTime = love.timer.getTime()
    math.randomseed(os.time())
    love.graphics.setBackgroundColor(0, 0, 0)

    -- Scene loading
    -- 加载一个你认为玩家应该第一个去到的场景
    local success, err = pcall(function()
        scenes.switchTo("scene_logo") -- Start with the logo scene.
        reset_room = "scene_logo" -- Which room will the player switch to after pressed f2.
    end)

    if (not success) then
        error(err)
    end
end
在这之后，每一个场景也可以看做是一个独立的环境了，模拟着main.lua的环境，转到update的地方： 
                
function love.update(dt)

    -- These are the libraries' update functions.
    keyboard.Update()
    sprites.Update(dt)
    typers.Update()
    tween.Update(dt)
    _CAMERA_:update(dt)

    -- 如果存在一个你选定的场景，那么update将持续锁定这个场景并且进行更新
    if (scenes.current) then
        if (scenes.current.update) then
            scenes.current.update(dt)
        end
    end

    -- The following code is used to limit the frame rate of the game.
    -- 这部分代码适用于手动控制帧数的
    frameTime = 1 / global:GetVariable("FPS")
    local endTime = love.timer.getTime()
    local elapsedTime = endTime - startTime
    if (elapsedTime < frameTime) then
        local sleepTime = frameTime - elapsedTime
        love.timer.sleep(sleepTime - 0.001)
        while (love.timer.getTime() - startTime < frameTime) do end
    end
    startTime = love.timer.getTime()

    -- The following code is used to update the audio manager.
    audio.Update()

end
在这之后，每个场景可能还需要有单独的绘制情况，所以继续看draw函数： 
                
function love.draw()

    -- 游戏全屏时候的硬算屏幕偏移及缩放情况
    screen_w, screen_h = love.graphics.getDimensions()
    scale = math.min(screen_w / 640, screen_h / 480)
    draw_x = math.floor((screen_w - 640 * scale) * 0.5 + 0.5)  -- 四舍五入到整数
    draw_y = math.floor((screen_h - 480 * scale) * 0.5 + 0.5)

    -- 设置主CANVAS
    love.graphics.setCanvas({CANVAS, stencil = true})
    love.graphics.clear(true, true)

    love.graphics.push()

        love.graphics.scale(scale, scale)

        -- 应用摄像机的偏移、旋转、缩放
        _CAMERA_:apply()
        love.graphics.setColor(1, 1, 1)

        if (scenes.current) then
            if (not scenes.current.PRIORITY) then -- 如果不是场景图层高
                if scenes.current and scenes.current.draw then
                    scenes.current.draw() -- 场景优先绘制
                end
                layers.sort() -- 其后所有贴图覆盖到上面
            else
                -- 反之
                layers.sort()
                if scenes.current and scenes.current.draw then
                    scenes.current.draw()
                end
            end
        end

        _CAMERA_:reset()

    love.graphics.pop()

    love.graphics.push()
        -- 绘制主CANVAS，这里也是你可以对整个屏幕施加shaders的地方
        love.graphics.setCanvas()
        love.graphics.translate(draw_x, draw_y)
        love.graphics.setColor(1, 1, 1)

        local shaders = global:GetVariable("ScreenShaders") or {}

        if (#shaders > 0) then
            local source = CANVAS
            local target = INTERMEDIATE_CANVAS

            for _, shader in ipairs(shaders) do
                love.graphics.setCanvas(target)
                love.graphics.clear()

                love.graphics.setShader(shader)
                love.graphics.draw(source)
                love.graphics.setShader()

                source, target = target, source
            end

            love.graphics.setCanvas()
            love.graphics.draw(source)
        else
            love.graphics.draw(CANVAS)
        end
    love.graphics.pop()
end
上面则是关于主文件对所有场景分支文件的处理方法。

屏幕的着色器（Shaders）

屏幕的着色器由一个全局变量控制： 

                
global:SetVariable("ScreenShaders", {})
如果这个表中含有任意一个shader元素，那么代码会尝试使用这个表里的元素加到主CANVAS上。
如果表为空，那么不会添加shaders。

                
local shader = love.graphics.newShader("Scripts/Shaders/gradient")
shader:send("color_tl", {1, 0, 0, 1})
shader:send("color_tr", {0, 0, 1, 1})
shader:send("color_br", {0, 1, 0, 1})
shader:send("color_bl", {1, 1, 0, 1})

local crtshader = love.graphics.newShader("Scripts/Shaders/CRT")
crtshader:send("scanlineSize", 4.0)
crtshader:send("scanIntensity", 0.8)
crtshader:send("scrollSpeed", 6.0)
crtshader:send("screenCurve", 0.5)

global:SetVariable("ScreenShaders", {crtshader, shader})
这会让整个屏幕拥有老电视机+渐变的两个shader叠加特效。
