    音频（Audio）¶

    用于播放音乐和音效。

    音乐的播放¶

    为了方便进行大音频的加载与流畅，LOVE2D允许使用者采用流式加载的方式来加载一个大音频，即加载一部分随后播放一部分。
    不至于有突然加载大量文件而导致严重卡顿影像体验。

    函数 audio.PlayMusic(music, volume, looping) -> Source, table 可以播放一个位于 Resources/Music/ 文件夹中的音频。

    示例： 

    -- 以 50% 的音量播放mus_gameover.ogg音频，保持循环状态
audio.PlayMusic("mus_gameover.ogg", 0.5, true)

    音效的播放¶

    音效大多都是比较短小的音频文件，LOVE2D也允许使用静态加载音频。

    函数 audio.PlaySound(sound, volume, looping) -> Source, table 用于播放一个位于 Resources/Sounds/ 文件夹中的音频。

    示例： 

    -- 以 80% 的音量播放snd_ding.wav音频，保持循环状态
audio.PlaySound("snd_ding.wav", 0.8)

    高阶使用技巧¶

    观前须知，你可能发现了上面的两个函数所返回的并不只是 table ，而还有一个 Source 。
    接下来 Source 将会被称为音频源或者源， table 会被称为管理器或管理表。

    通常情况下，只使用一个变量来获取音频流返回的内容，是无法获取到管理器的，而管理器只被应用于高阶技巧或修改底层。

    -- 只能获取源
local mus = audio.PlayMusic("a.ogg", 1, true)

-- 可以获取源和管理器
local mus, ins = audio.PlayMusic("a.ogg", 1, true)

    音量的渐变¶

    使用 VolumeTransition(begin, target, duration) 方法来动态改变一个播放流的音量渐变效果。

    示例： 

    -- 创建一个音频流，并用mus和ins分别承接源和管理器
local mus, ins = audio.PlayMusic("mus_gameover.ogg", 0, true)

-- 只有身为管理器的ins可以使用这个渐变函数，从0开始一直到1，渐变过程持续1.4秒
ins:VolumeTransition(0, 1, 1.4)

    音调的渐变¶

    使用 PitchTransition(begin, target, duration) 方法来动态改变一个播放流的音量渐变效果。

    由于其结构和上面 VolumeTransition 的结构完全一致，这里不再赘述。

    设置循环区间¶

    使用 SetLoopPoints(startTime, endTime) 方法来给一个音频源添加循环点。

    当时间到达endTime的时候，会自动回到startTime的地方，制造一个循环段落。如果要关闭循环则需要令 ins.loopStart 为 false 。

    设置音频效果¶

    使用 SetEffect(effect_name, params) 来设置一个音频源的音频效果。

    需注意：某些设备并不支持添加音频效果，所以音频可能没有任何改变。

    音频效果¶

    接下来是所有可用的音频效果：
    首先让我们排除 effect_name 参数（也就是第一个参数），因为第一个参数写什么字符串都是可以的。

    注意：如果你不想修改这些参数，或者不知道需不需要修改这些参数，你可以直接填写每个效果最后的英文，需要注意是全小写即可。 

    local myreverb = {
    type = "reverb"
}
ins:SetEffect("myr", myreverb)

    混响（Reverb）¶

    用来模拟声音在不同空间环境下的反射效果（比如大厅，洞穴）。

    • (number) gain：整体增益（音量），默认0.316（约 -10 dB）。
    • (number) gainhf：高频增益衰减因子（相对于低频），默认0.89。
    • (number) room：房间大小影响的基础增益，默认0.5。
    • (number) roomhf：房间大小对高频的影响因子，默认0.5。
    • (number) decay：混响衰减时间（秒），默认1.49。
    • (number) decayhf：高频的衰减时间（相对于低频），默认0.83。
    • (number) reflectionsgain：早期反射声的增益，默认0.05（约 -26 dB）。
    • (number) reflectionsdelay：早期反射声的延迟时间（秒），默认0.007。
    • (table) reflectionspan：早期反射声的三维方向向量，默认为 {0.0, 0.0, 0.0}，表示来自所有方向。
    • (number) lategain：晚期混响（混响声尾）的增益，默认1.26（约 2 dB）。
    • (number) latedelay：晚期混响的延迟时间（秒），默认0.011。
    • (table) latepan：晚期混响的三维方向向量，默认为 {0.0, 0.0, 0.0}，表示来自所有方向。
    • (number) diffusion：混响声的扩散度，影响密度感，默认1.0。
    • (number) density：混响声的密度，默认1.0。
    • (number) hfreference：计算高频衰减的参考频率（Hz），默认5000.0。
    • (number) lfreference：计算低频衰减的参考频率（Hz），默认250.0。
    • (number) roomrolloff：房间大小随距离增加的衰减因子，用于模拟距离感，默认0.0。
    • (number) airborneabsorbtiongainhf：高频在空气中的吸收衰减因子，默认0.994（约 -0.05 dB）。

    合唱（Chorus）¶

    通过混合延迟的声音副本来产生合唱效果，模拟多个声源同时发声。

    • (number) rate：调制频率（Hz），控制效果快慢，默认1.1。
    • (number) depth：调制深度，控制音高变化的范围，默认0.1。
    • (number) feedback：反馈量，将处理后的声音送回输入端，默认0.25。
    • (number) delay：基础延迟时间（秒），默认0.016。
    • (table) pan：效果声的三维方向向量，默认为 {0.0, 0.0, 0.0}，表示来自所有方向。

    失真（Distortion）¶

    通过剪切波形来产生过载、失真的音效。

    • (number) gain：失真前的输入增益，默认0.05。
    • (number) edge：失真边缘，影响失真度的“硬度”，默认0.2。
    • (number) posteqcenterfrequency：失真后均衡的中心频率（Hz），默认2400.0。
    • (number) posteqbandwidth：失真后均衡的带宽（Hz），默认2400.0。
    • (number) prelowpasscutoff：失真前低通滤波器的截止频率（Hz），默认8000.0。

    回声（Echo）¶

    模拟声音在空间中的重复反射。

    • (number) delay：两次回声之间的延迟时间（秒），默认0.1。
    • (number) lrdelay：左右声道回声的延迟差（秒），用于立体声效果，默认0.1。
    • (number) damping：每次回声的高频衰减因子，默认0.5。
    • (number) feedback：反馈量，控制回声持续的时间，默认0.5。
    • (number) spread：回声在立体声场中的扩散度，默认-1.0。

    均衡器（Equalizer）¶

    通过调整不同频段的增益来改变音色。

    • (number) lowgain：低频增益，默认1.0。
    • (number) lowcutoff：低频截止频率（Hz），默认200.0。
    • (number) mid1gain：中低频增益，默认1.0。
    • (number) mid1center：中低频的中心频率（Hz），默认500.0。
    • (number) mid1width：中低频的带宽，默认1.0。
    • (number) mid2gain：中高频增益，默认1.0。
    • (number) mid2center：中高频的中心频率（Hz），默认3000.0。
    • (number) mid2width：中高频的带宽，默认1.0。
    • (number) highgain：高频增益，默认1.0。
    • (number) highcutoff：高频截止频率（Hz），默认6000.0。

    镶边（Flanger）¶

    通过混合一个轻微延迟且延迟时间周期性变化的信号来产生旋转、飘渺的效果。

    • (number) rate：调制频率（Hz），控制效果快慢，默认0.1。
    • (number) depth：调制深度，默认1.0。
    • (number) feedback：反馈量，默认-0.5。
    • (number) delay：基础延迟时间（秒），默认0.002。

    环形调制器（Ring Modulator）¶

    通过将输入信号与一个内部振荡器信号相乘来产生金属感、机器人声效。

    • (number) frequency：内部振荡器的频率（Hz），默认440.0。
    • (number) highpasscutoff：输出信号的高通滤波器截止频率（Hz），默认800.0。
    • (number) waveform：内部振荡器的波形类型。0为正弦波，1为方波，默认为0。

    音高偏移（Pitch Shifter）¶

    实时改变声音的音高而不改变播放速度。

    • (number) coarsetune：粗调音高，以半音为单位，默认0。
    • (number) finetune：细调音高，以音分为单位，默认0。

    压缩器（Compressor）¶

    通过降低大信号、提升小信号来减少动态范围，使音量更平均。

    • (number) onoff：开关。1为开启，0为关闭，默认0。
