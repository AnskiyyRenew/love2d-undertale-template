    文本（Typers）¶

    如果要做出打字机的效果，或者带标签诸如bbcode的效果，请看这个页面的介绍。

    文本的创建方法¶

    SoulEngine 中拥有两种类型的文本，一种是 打字机 样式的，另一种是 直接绘制 样式的。

    下面介绍这两种不同类型打字机的创建方法：

    typers.CreateText(sentences, position, layer, bubblesize, progressmode) -> table¶

    此函数用于创建一个逐字打字效果的打字机。

    • sentences 要打的字，可以使用字符串或一个表。
    • position 打在什么地方，格式为 {x, y} 。
    • layer 图层大小。
    • bubblesize 气泡尺寸大小，格式为 {w, h} 。
    • progressmode 打字机模式，在默认情况下， progressmode 不加以设置则使用 none 模式。目前拥有三个模式：
      • manual 手动模式。需要手动操作才会推进打字机的进程。
      • auto 自动模式。在每次句子打完之后都会停留3秒，之后自动推进句子，无法手动干涉。
      • none 代码模式。永远无法手动干涉，只能通过代码层面进行控制。
    -- 这个打字机使用表来设置要打的句子。
local t = typers.CreateText({
    "这是第一句话",
    "这是第二句话"
}, {320, 240}, 1.1, {0, 0}, "manual")

-- 这个打字机使用字符串来设置要打的句子。
local t = typers.CreateText("这是一句话", {320, 240}, 1.1, {0, 0}, "manual")

    typers.DrawText(text, position, layer) -> table¶

    此函数用于直接绘制文本，跳过打字部分。

    • text 要打什么文本。
    • position 打在什么位置。
    • layer 图层。
    -- 将START打在(320, 240)的位置，图层为1.
local t = typers.DrawText("START", {320, 240}, 1)

    DrawText 的颜色说明¶

    使用 typers.DrawText 创建的直接绘制文本时，修改颜色不会即时生效——你必须先设置对象的 color 字段，然后调用 :Reparse() 使更改生效。例如：

    local t = typers.DrawText("Hello", {320, 240}, 1)
-- 将颜色改为红色（RGB）
t.color = {1, 0, 0}
-- 重新解析以使颜色生效
t:Reparse()

    如果你想动态地改变整段文本的透明度或颜色，同样需要在修改对应字段后调用 :Reparse()（或直接修改 parsed 中每个片段的 color 字段）。

    文本的标签¶

    打标签的能力用于在文本中嵌入格式控制（颜色、字体、缩放、偏移等）。标签使用中括号形式写入，例如 [red]、[font=simsun.ttc,13] 等，会影响后续字符直到被新的标签覆盖。

    常见标签示例：

    • [colorName]：使用预设颜色名（如 red, green, blue, yellow, white, black 等），例如 [red]文本。
    • [font=字体名,大小]：切换字体与字号，例如 [font=simsun.ttc,13]。
    • [scale=数字]：缩放字符，例如 [scale=2] 会把后续字符放大为 2 倍。
    • [spaceX=数字]：设置字符间距，例如 [spaceX=10]。
    • [offsetX=数字] / [offsetY=数字]：设置字符的绘制偏移。
    • [preset=名称]：使用预设（实现中存在 selection、chinese、chd 等预设），会一次性改变字体、大小、偏移、缩放等属性。

    标签是由 typers.DrawText 在解析阶段读取并应用的；如果你在运行时修改了 rawtext，注意要调用 :Reparse() 以重新解析并生成 parsed 数据。

    方法¶

    下面列出 typers.CreateText（打字机）和 typers.DrawText（直接绘制）实例可调用的方法，说明参数与典型用法。

    CreateText 实例方法（元表 functions）¶

    • :Reset()：重置打字器状态（清除已出现的字、恢复默认速度、字体、颜色等）。
    • :GetLettersSize() -> width, height：返回当前已生成字块所占宽高（使用 typer.texturelen 计算）。
    • :ShowBubble(direction, position)：显示对话气泡并将尾巴指向 direction（"left"/"right"/"up"/"down"），position 为尾巴在气泡边上的相对位置（数值）。
    • :HideBubble()：隐藏当前气泡（将所有气泡元素 alpha 设为 0）。
    • :SetText(texts, instant)：设置要显示的句子，texts 可为字符串或字符串数组；instant 为 true 时会进入跳过模式并立即填充当前句子。
    • :Destroy()：销毁该实例并释放相关资源（移除对象、释放字体、删除字数据等）。
    • :Remove()：同 :Destroy() 的别名。

    另外，CreateText 实例暴露两个回调： - OnUpdating：每次生成新字母前调用（可用于同步外部逻辑）。 - OnComplete：所有句子完成后调用。

    示例：

    local t = typers.CreateText({"这是句子"}, {320,240}, 1.1, {0,0}, "manual")
-- 手动显示气泡并在右侧 0.2 位置指向
t:ShowBubble("right", 0.2)
-- 更换句子并立即填充
t:SetText("新句子", true)
-- 销毁
t:Destroy()

    DrawText 实例方法（元表 functions_instant）¶

    • :GetLettersSize() -> width, height：计算并返回解析后文本的宽高。
    • :SetText(text)：设置 rawtext 并立即调用 :Reparse() 解析文本（用于动态更新内容）。
    • :MoveTo(x, y)：移动文本位置并在需要时重新解析（保持排版正确）。
    • :Destroy()：销毁实例并释放字体缓存与解析数据。
    • :Remove()：同 :Destroy() 的别名。

    示例：

    local d = typers.DrawText("START", {320,240}, 1)
-- 改变文本并生效
d:SetText("新的内容")
-- 移动位置
d:MoveTo(100, 200)
-- 获取尺寸
local w, h = d:GetLettersSize()
-- 销毁
d:Destroy()

    注意：如果你直接修改 DrawText 实例的 color、alpha 等字段，需要调用 :Reparse()（或使用 :SetText()/赋值 rawtext 触发的 Reparse）才能使颜色/透明度等在渲染中生效。

    CreateText 的标签¶

    typers.CreateText 支持更丰富的标签（以 [...] 形式），可在逐字打字过程中改变字体、颜色、暂停、播放音效、调用函数等。以下为实现中解析到的标签及作用：

    • colorRGB:r,g,b：按 RGB 直接设置颜色（数值 0-1），例如 [colorRGB:1,0,0]。
    • colorHEX:rrggbb：按十六进制设置颜色，例如 [colorHEX:FF0000]。
    • fontIndex:n：切换到字体列表中的第 n 个字体。
    • font:name：将当前字体替换为指定字体名（字符串）。
    • fontSize:n：设置当前字体大小；可写为 index,size 来设置指定索引字体的大小。
    • alpha:n：设置当前透明度（0-1）。
    • offset:index,x,y：设置指定字体索引的偏移 {x,y}。
    • voice:name：切换打字音为指定音效文件（voice 列表）。
    • sound:name[,islooping]：播放指定音效，第二个参数可选表示是否循环。
    • space:index,number：设置字符间距（positions.spacing）。
    • wait:n：暂停 n 帧再继续打字（仅暂停当前字）。
    • waitALL:n：将打字间隔与 sleep 一并设置为 n（整段等待）。
    • waitKEY:key：等待指定按键按下才继续（会进入等待状态）。
    • scale:n：设置缩放倍数。
    • speed:n：调整速度（内部通过设置 sleep 值影响打字速率）。
    • effect:name,intensity：设置字母出现时的特效（如 shake、rotate）及强度。
    • function:funcName|param1,param2,...：调用全局函数，可带参数（逗号分隔）。
    • pattern:chinese|english：快速切换中/英文排版模式（字体索引、缩放与间距）。
    • RESET:what 或 RESET：重置指定属性（color、font、fontSize、alpha、position），或全部重置。

    另外还有若干单词型标签（无需冒号）：

    • skip / instant：跳过当前句子剩余字符并进入跳过模式。
    • shadow / noshadow：开启/关闭阴影绘制。
    • noskip：禁止跳过功能。
    • voice / novoice：开启/关闭打字声。
    • canskip：允许跳过。
    • next：直接跳到下一句（若已是最后一句则结束）。

    示例：

    local t = typers.CreateText({"[colorRGB:1,0,0]红色字[RESET] 普通字"}, {320,240}, 1.1, {0,0}, "manual")

    标签在解析阶段被立即应用，影响后续字符；某些标签（如 waitKEY）会使打字器进入等待状态直到条件满足。
